## 🎯 产品概述

### 1.1 产品定位
> **Shaker - 智能鸡尾酒推荐专家，让每个人都能找到完美的那一杯**

- [x] **核心价值主张**：通过AI驱动的个性化推荐系统，根据用户场景、心情、可用材料和口味偏好，智能推荐最适合的鸡尾酒配方
- [x] **目标用户群体**：鸡尾酒爱好者、家庭调酒新手、酒吧从业者、社交聚会组织者
- [x] **解决的核心问题**：解决选择困难、配方复杂、材料搭配难、缺乏专业知识等调酒痛点
- [x] **技术实现方式**：基于Web技术栈，集成Coze AI API，实现智能推荐算法

### 1.2 产品目标
- [x] **市场定位目标**：成为鸡尾酒推荐领域的"智能且懂你"代名词
- [x] **用户增长目标**：MVP版本验证产品可行性，获得初期用户反馈
- [x] **产品差异化目标**：相比传统配方网站，提供AI驱动的个性化体验和情境化推荐
- [x] **商业验证目标**：验证"AI+鸡尾酒推荐"商业模式的市场接受度
- [x] **效率提升目标**：帮助用户在30秒内获得3-5个精准推荐，节省90%的选择时间
- [x] **成果可视化目标**：推荐准确度高，用户满意度显著提升，复用率增长
- [x] **使用体验目标**：零学习成本上手，3步完成推荐，一键获取完整配方
- [x] **技术稳定性目标**：API响应时间<15秒，推荐成功率≥95%，错误恢复能力强
- [x] **性能优化目标**：页面加载<3秒，操作响应<200ms，网页端体验流畅
- [x] **兼容性目标**：支持主流浏览器，适配多种设备尺寸，跨平台一致体验
- [x] **扩展性目标**：预留用户系统、收藏功能、社交分享等后续功能接口（mvp版本不考虑）

### 1.3 MVP版本范围
- [x] **包含功能**：
  - [x] **场景选择功能**：6种主要使用场景，卡片式选择交互
  - [x] **心情描述功能**：6种森林主题心情状态，支持多选标签
  - [x] **材料选择功能**：完整的酒类和配料分类，智能搜索
  - [x] **偏好设置功能**：酒精度、甜度、酸度等滑块调节
  - [x] **AI推荐功能**：基于Coze API的智能推荐算法
  - [x] **结果展示功能**：详细配方、制作步骤、视觉效果展示

#### ❌ 不包含功能
- [x] **用户账号系统**：暂不支持用户注册登录和个人数据保存
- [x] **历史记录功能**：暂不保存用户的推荐历史和收藏
- [x] **社交分享功能**：暂不支持分享到社交媒体平台
- [x] **评价反馈系统**：暂不收集用户对推荐结果的评价
- [x] **多语言支持**：暂只支持中文界面
- [x] **功能简化**：聚焦核心推荐流程，剔除复杂的高级功能
- [x] **界面简化**：采用极简设计，只保留必要的输入项和展示内容
- [x] **流程简化**：从"输入→推荐→展示"的直线流程，无复杂分支
- [x] **技术简化**：使用成熟稳定的技术栈，避免过度工程化

### 2.2 用户使用流程

#### 🚀 核心使用路径
```
用户行为路径：
[触发场景] → [产品交互] → [预期结果]

具体路径：
[选择场景] → [想要调制鸡尾酒但不知道做什么/想喝鸡尾酒但不知道哪个适合自己/有部分材料但不知道可以做哪些/想要给别人准备合适的鸡尾酒] → [打开Shaker输入自己/别人的场景、心情、材料、偏好等详细需求] → [获得个性化推荐和完整配方/获得基于现有材料的创意推荐/获得适合氛围的多样化选择]

```

#### 📅 典型使用场景

**场景1：新手入门**
```
想尝试调制鸡尾酒但毫无经验
    ↓
打开Shaker → 选择"独处放松"场景 → 选择"平静"心情
    ↓
输入家中现有材料 → 设置"低酒精度"偏好 → 点击获取推荐
    ↓
获得3个简单易做的推荐 → 选择一个查看详细步骤 → 成功制作第一杯鸡尾酒
```

**场景2：聚会策划**
```
为朋友聚会准备合适的鸡尾酒
    ↓
选择"聚会派对"场景 → 选择"春日愉悦+夏日激情"心情组合
    ↓
输入采购清单中的材料 → 设置"中酒精度"和"好看颜值"偏好
    ↓
获得5个适合聚会的推荐 → 选择2-3个准备材料 → 聚会成功
```

**场景3：约会浪漫**
```
想为约会之夜准备特别的鸡尾酒
    ↓
选择"浪漫约会"场景 → 选择"春日愉悦"心情 → 添加"Instagram风格"特殊要求
    ↓
设置偏好为"微甜+清爽+彩色" → 获取推荐
    ↓
获得颜值超高的浪漫酒款 → 跟随步骤制作 → 约会氛围满分
```

#### 💡 主要用户 vs 次要用户

**🎯 主要用户** - 调酒新手 & 聚会主办者 & 酒吧客人 
- **痛点**：不知道该做什么酒，材料搭配复杂，缺乏经验，缺乏进一步沟通（酒吧一般只提供酒单）
- **期望**：推荐准确，步骤清晰，材料容易获得
- **行为特征**：重视简单易懂，关注成功率和效果展示

**🔄 次要用户** - 浪漫情侣 & 居家爱好者  
- **痛点**：寻求创新和个性化，希望有独特体验
- **期望**：推荐有创意，颜值高，符合特定情境
- **行为特征**：更关注个性化和差异化，愿意尝试复杂配方

#### 💎 Shaker的用户价值主张

**🎯 核心价值**
1. **智能个性化** - AI理解你的需求，推荐最适合的选择
2. **场景化推荐** - 不只是配方，更是适合当下情境的完美选择
3. **零门槛上手** - 3步获得推荐，小白也能调出专业水准
4. **完整指导** - 从材料到步骤，从口感到颜值的全方位指导

### 3.2 核心业务流程

#### 🎯 主流程：从输入到推荐

**阶段1：用户输入收集**
```
用户进入首页
    ↓
显示欢迎界面和开始按钮
    ↓
点击开始，进入输入流程：
├── 步骤1：场景选择（6选1，卡片式）
├── 步骤2：心情描述（7选N，标签式）
├── 步骤3：材料选择（分类多选，搜索支持）
├── 步骤4：偏好设置（滑块+选择器）
└── 步骤5：特殊要求（可选文本输入）
```

**阶段2：AI处理阶段**
```
用户点击"获取推荐"按钮
    ↓
前端数据验证和格式化
    ↓
显示加载动画和等待提示
    ↓
后端处理：
├── 收集所有用户输入数据
├── 构建结构化的推荐Prompt
├── 调用Coze API获取AI推荐
├── 解析AI返回的推荐内容
└── 格式化为标准数据结构
```

**阶段3：结果展示阶段**
```
AI处理完成
    ↓
隐藏加载状态，显示推荐结果
    ↓
推荐结果界面：
├── 推荐列表展示（3-5个选项）
├── 每个推荐包含：名称、理由、难度、预览
├── 点击查看详情：完整配方和制作步骤
├── 支持重新推荐或修改输入
└── 提供分享功能（后续版本）
```

**阶段4：用户反馈循环**
```
用户查看推荐结果
    ↓
可选操作：
├── 满意结果 → 查看详细配方 → 开始制作
├── 不满意 → 点击重新推荐 → 获得新结果
├── 需要调整 → 返回修改输入 → 重新生成
└── 想要更多 → 调整偏好设置 → 获得变化
```
